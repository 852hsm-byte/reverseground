ReverseGround 시스템 가이드

option-a-proctabs.html 기반 랜딩페이지 디자인 규칙. 수정·추가 시 이 문서를 기준으로 일관성을 유지하세요.

파일 구조

reverseground-v2/
├── option-a-proctabs.html ← 메인 페이지
├── components/
│   └── gnb.js ← GNB (HTML + CSS + JS 통합)
└── css/
     ├── tokens.css ← 디자인 토큰 ★ 가장 먼저 로드
     ├── base.css ← 리셋, 공통 레이아웃, 버튼
     ├── hero.css ← 히어로 + Intro strip
     ├── services.css
     ├── platform-tabs.css
     ├── new-sections.css ← Procsy 탭 내부 섹션
     ├── features.css
     ├── about.css · vision.css · howto.css · pricing.css · contact.css · footer.css
     ├── modern-proctabs.css ← responsive.css 이후 로드 (탭 스타일 최종 우선)
     └── responsive.css ← 반응형 오버라이드 ★ 항상 마지막 로드
⚠️
CSS 로드 순서 주의: modern-proctabs.css는 responsive.css보다 나중에 로드됩니다. Procsy 탭·패널 관련 스타일은 반드시 modern-proctabs.css에서 수정할 것.

핵심 규칙

  • 🎨
    색상·폰트크기·border-radius는 raw 값 직접 사용 금지 — 반드시 tokens.css의 var(--*)만 사용. (0, 50%는 구조적 키워드라 예외)
  • 📐
    반응형 오버라이드는 반드시 css/responsive.css에만 작성 (항상 마지막 로드). Procsy 탭/패널 예외 → modern-proctabs.css
  • 🖱️
    hover 효과는 반드시 @media (hover: hover) and (pointer: fine) 안에서만. 터치 리셋은 @media (hover: none) 안에.
  • ⚫
    Pure black #000 사용 금지 — 텍스트는 var(--neutral-900) #191919 사용. 더 진하면 var(--near-black) #111111
  • ✏️
    새 토큰이 필요하면 현재 HTML 완성본 기준으로 값을 tokens.css에 추가 후 var()로 참조. 비슷한 기존 토큰으로 대체해 결과물을 바꾸지 말 것.
  • 💅
    rgba(...) 투명도 오버레이는 토큰화 대상 제외 — raw 값 그대로 사용 허용 (183곳·64개 조합, 구조 변경 비용 대비 실익 없음)
  • 🔗
    GNB는 components/gnb.js가 주입 — HTML 파일에서 직접 수정 금지
